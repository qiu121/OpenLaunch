# OpenLaunch PRD

文档状态：MVP 实施中  
日期：2026-07-05  
阶段：产品需求与技术可行性分析  
实现状态：已进入本机 Developer Build 编码实施

## 1. 背景

新版 macOS 已经移除传统 Launchpad，系统提供的 Apps / Spotlight 应用视图可以承担基础启动功能，但在“全屏图标网格、自定义排序、分页布局、类 Launchpad 的肌肉记忆”上不完全等价。

OpenLaunch 的目标是提供一个轻量、可控、接近旧 Launchpad 使用习惯的 macOS 应用启动器。它不追求复杂的应用管理平台，而是优先解决一个具体问题：用户按下快捷键后，能快速看到所有常用 app，并以自己喜欢的顺序打开它们。

## 2. 产品定位

OpenLaunch 是一个 macOS 全屏应用网格启动器。

核心价值：

- 类 Launchpad 的全屏图标网格体验。
- 支持应用排序，包括名称、添加时间、最近打开、自定义排序。
- 支持分页显示或滚动显示。
- 轻量、快速、少设置、少打扰。

产品原则：

- 简单优先，不复制完整 Launchpad。
- 优先本机体验，不依赖云服务。
- 优先公开系统 API，不使用脆弱的私有接口。
- 优先可维护，不一开始做过多主题、同步、文件夹等扩展能力。

## 3. 目标用户

### 3.1 主要用户

使用 macOS 26+，习惯旧 Launchpad，全屏打开 app 的用户。

典型需求：

- 不想每次用 Spotlight 搜索。
- 希望看到图标网格。
- 希望按自己的顺序排列 app。
- 希望有一个快捷键快速呼出启动器。

### 3.2 次要用户

拥有大量应用、希望用排序和布局提升启动效率的高级用户。

典型需求：

- 根据安装时间或最近打开时间找 app。
- 手动整理常用 app。
- 在分页和滚动两种布局之间切换。

## 4. MVP 范围

MVP 只做以下功能：

1. 全屏/近全屏启动器窗口。
2. 扫描本机已安装应用。
3. 展示应用图标和名称。
4. 点击图标启动应用。
5. 支持排序：
   - 名称排序。
   - 添加时间排序。
   - 最近打开排序。
   - 自定义排序。
   - 名称、添加时间、最近打开支持正序/倒序；自定义排序按用户拖拽顺序展示。
6. 支持显示模式：
   - 分页网格。
   - 垂直滚动网格。
7. 支持基础搜索。
8. 支持本地保存设置和布局。
9. 支持快捷键显示/隐藏窗口。

## 5. 非目标

MVP 不做：

- 不恢复系统 Launchpad。
- 不修改 macOS 自带 Apps / Spotlight 视图。
- 不接管系统手势。
- 不做 iCloud 同步。
- 不做文件夹。
- 不做复杂主题市场。
- 不做自动更新。
- 不做 App Store 发布。
- 不做应用卸载、清理、权限扫描等管理功能。
- 不使用私有 API 模拟系统 Launchpad。
- 不做对外发布所需的 Developer ID 签名和 Apple 公证。

这些能力可以作为后续版本评估，但不进入第一版。

## 6. 使用流程

### 6.1 首次启动

1. 用户打开 OpenLaunch。
2. 应用扫描系统中的 `.app` bundle。
3. 主界面显示应用网格。
4. 默认排序为添加时间旧到新，新添加的软件显示在最后。
5. 用户可通过主界面设置按钮调整显示模式和排序；排序类型直接显示在设置菜单中，需要方向的类型再展开具体顺序。

### 6.2 日常使用

1. 用户按下快捷键。
2. OpenLaunch 显示全屏网格。
3. 用户点击 app 图标。
4. OpenLaunch 启动目标 app。
5. 启动成功后，OpenLaunch 自动隐藏。
6. OpenLaunch 记录最近打开时间；通过 OpenLaunch、Dock 或其它入口激活的已扫描应用都应进入记录。

### 6.3 自定义排序

1. 用户通过主界面设置按钮，在排序分组中直接选择“自定义”。
2. 用户拖动 app 图标调整顺序。
3. OpenLaunch 立即保存布局。
4. 下次打开时恢复该顺序。

## 7. 功能需求

### 7.1 应用扫描

扫描路径：

- `/Applications`
- `/Users/<user>/Applications`，文档中可简写为 `~/Applications`
- `/System/Applications`
- `/System/Cryptexes/App/System/Applications`
- `/Applications/Utilities`
- `/System/Library/CoreServices/Applications`

默认扫描策略：

- 优先使用 Spotlight 应用索引获取候选 `.app`，补齐 LaunchOS/Launchpad 类工具能看到的系统公开应用。
- 同时保留文件系统递归扫描作为兜底，避免 Spotlight 索引异常时列表为空。
- 递归监听默认扫描目录的文件系统变化；检测到应用安装、删除、移动或 `.app` 内部写入完成后，在防抖后后台重新扫描。
- 自动刷新发现新增应用后，下次打开启动台时新增图标使用一次性轻弹入场动画；分页模式会定位到第一个新增应用所在页；删除应用不展示退场动画。
- 不使用定时轮询作为主刷新机制，避免后台空转和重复扫描。
- Spotlight 结果使用用户可见路径策略：公开应用目录、用户目录中的独立 `.app`、少量共享支持目录可进入候选；系统 framework、daemon、updater、模板 app 等内部 bundle 不进入候选。
- 系统 Cryptex 应用目录必须纳入扫描，Safari 等系统应用可能通过 `/Applications` 符号链接指向 Cryptex 中的真实 bundle。
- 扫描结果保存真实 bundle 路径，避免 `NSWorkspace` 对符号链接应用图标叠加别名角标。
- `LSUIElement` 不再一刀切过滤，Docker、Tailscale、Screenshot、System Information、LaunchOS 等有 UI 或菜单栏入口的应用应可显示。

可选补充路径：

- 浏览器创建的 Web App 目录。
- 用户自定义扫描目录。

每个应用需要读取：

- 应用路径。
- bundle identifier。
- 显示名称，优先使用系统本地化展示名。
- 图标。
- 分类信息。
- 文件创建时间。
- 文件修改时间。
- Spotlight 添加时间，如果可用。
- Spotlight 展示名称，如果可用。

过滤规则：

- 过滤 `LSBackgroundOnly` 后台应用。
- 过滤明显不可直接启动的系统组件。
- 过滤重复 bundle identifier 或重复路径。
- 对无法读取 Info.plist 的项目降级处理。

### 7.2 应用图标

需求：

- 使用系统 API 获取 app 图标。
- 图标大小跟随网格密度变化。
- 图标下方显示应用的系统本地化名称。
- 名称过长时最多显示两行，避免溢出。
- 支持图标缓存，减少每次打开时的加载延迟。

图标风格：

- 主 app 图标可采用“3x3 应用网格 + 打开/启动符号”的设计。
- 避免直接复制 Apple Launchpad 图标。
- Dock/app 入口图标与菜单栏状态项图标分离：Dock 图标强调官方应用入口语义，状态栏图标保持轻量网格符号。

### 7.3 启动应用

需求：

- 点击图标启动 app。
- 启动成功后隐藏 OpenLaunch。
- 启动失败时显示轻量错误反馈。
- 记录最近打开时间。

失败场景：

- app 已被删除。
- app 路径变化。
- 权限不足。
- bundle 损坏。

处理方式：

- 启动失败时提示“无法打开该应用”。
- 提供“重新扫描”入口。
- 不阻塞整个启动器。

### 7.4 排序

支持排序方式：

1. 添加时间排序。
2. 名称排序。
3. 最近打开排序，未打开的应用在前，最近打开时间越新的应用越靠后。
4. 自定义排序，通过主界面设置按钮进入后拖动图标调整。

排序顺序：

- 添加时间、名称、最近打开均支持正序和倒序。
- 添加时间正序文案为“最早添加”，倒序文案为“最近添加”。
- 名称正序文案为“A 到 Z”，倒序文案为“Z 到 A”。
- 最近打开正序文案为“最早打开”，倒序文案为“最近打开”。
- 自定义排序不显示、不应用排序顺序，始终尊重用户拖拽后的布局。
- 主界面设置在菜单根层的排序分组中直接显示“添加时间 / 名称 / 最近打开 / 自定义”；添加时间、名称、最近打开再右侧展开具体顺序，用户点击具体顺序后菜单关闭。

添加时间说明：

- macOS 不保证所有 app 都有准确安装时间。
- 首次建档优先使用可信的 Spotlight 添加时间。
- Spotlight 时间早于文件创建时间时视为异常，并使用文件创建时间。
- 无法读取时依次降级使用文件创建时间、文件修改时间。
- 建档后保存每个应用的添加时间；自动更新不改变原有顺序，新发现应用记录当前发现时间。
- UI 文案建议使用“添加时间”，而不是“安装时间”。

实现约束：

- 添加时间首次建档应优先使用经过校验的 `kMDItemDateAdded`，不能持续用易变化的系统元数据覆盖本地记录。
- 不能只依赖 `.app` bundle 的创建时间，因为部分应用更新或迁移后创建时间会早于实际添加时间。
- 相同添加时间使用名称作为稳定次级顺序；无法确认添加时间的应用在正序和倒序中均排在已知时间之后。
- 默认排序方向为旧到新，新添加的软件显示在最后。
- 排序顺序文案使用“最早添加 / 最近添加”等业务语言，避免技术化的“升序 / 降序”。

自定义排序：

- 保存用户拖拽后的顺序。
- 以 bundle identifier + 路径作为稳定识别依据。
- app 删除后保留布局记录一段时间，重新安装后可尝试恢复。
- 搜索过滤状态下不进入拖拽排序，避免用户只重排部分搜索结果造成误操作。
- 拖拽到目标位置前应显示类似 Dock 图标重排的横向插入槽预览；被拖拽或刚操作过的 app 图标不应出现变暗状态。

### 7.5 显示模式

分页模式：

- 接近旧 Launchpad。
- 固定行列数。
- 底部只显示分页圆点，不显示页码文本。
- 分页圆点必须可点击跳转。
- 支持左右滑动手势翻页，鼠标/触控板横向滚轮的小幅连续输入需要形成跟手的中间页状态，并在释放后稳定翻到目标页。
- 分页采用交互式分页：拖动期间页面轨道连续跟随指针，释放后按页面进度回落到当前页或进入相邻页，避免一次滑动越过多页。
- 支持键盘方向键翻页。
- 分页控件不得遮挡应用图标点击区域。
- 页面切换使用整页横向轨道滑移，不使用内容瞬间替换或单页进出场过渡。
- 横向拖拽期间页面轨道必须跟随指针移动，首尾页越界拖拽使用轻微阻尼。
- Dock 或其他应用被激活时，OpenLaunch 覆盖层必须自动隐藏，让目标 app 进入前台。
- 进入 OpenLaunch 全屏覆盖层后，顶部菜单栏默认并持续隐藏；鼠标贴近顶部不再触发菜单栏浮出，避免状态栏占位条、闪烁或与 Dock 浮出同步。

滚动模式：

- 垂直滚动。
- 适合 app 数量多的用户。
- 实现更简单。

设置项：

- 显示模式：分页 / 滚动。
- 网格密度：小 / 中 / 大。
- 是否显示应用名称。
- 是否显示搜索框。

### 7.6 搜索

MVP 搜索需求：

- 全屏界面显示时，搜索框不默认显示输入光标。
- 用户点击搜索框或直接输入文字时，输入必须进入搜索框并显示输入光标。
- 每次重新进入 OpenLaunch 时应重置上次的搜索文本和输入光标状态。
- 输入时实时过滤应用。
- 匹配本地化应用名称。
- 匹配原始应用名称和 `.app` 文件名。
- 匹配 bundle identifier。
- 搜索结果按当前排序方式展示。
- 搜索实时变化，不需要按回车确认。
- Esc 优先退出全屏界面。

暂不做：

- 拼音搜索。
- 模糊缩写搜索。
- 应用内动作搜索。
- 文件搜索。

### 7.7 快捷键

需求：

- 用户可以设置一个全局快捷键。
- 默认建议使用不容易冲突的组合键。
- 呼出时显示 OpenLaunch。
- 再次触发时隐藏 OpenLaunch。

技术注意：

- 某些系统快捷键不可接管。
- `Fn-Shift-A` 可能被系统 Apps 视图占用，不建议默认使用。
- 全局快捷键可能需要辅助功能权限，取决于实现方式。

### 7.8 全屏与退出

需求：

- OpenLaunch 使用覆盖式全屏窗口，铺满鼠标所在屏幕。
- 不显示普通 macOS 标题栏和红黄绿窗口按钮。
- 用户不应明显感知到这是普通第三方窗口。
- 全屏覆盖时不显示顶部系统菜单栏整块区域，顶部滑出也不显示菜单栏；OpenLaunch 菜单栏状态项作为退出覆盖层后的启动入口保留。
- 覆盖窗口不得压住 Dock；常显 Dock 应可见，自动隐藏 Dock 应能从屏幕边缘触发浮出。
- 自动隐藏 Dock 场景下不得常态露出明显桌面底边；Dock 应通过系统层级自然浮出到 OpenLaunch 之上。
- Dock 浮出与顶部菜单栏必须解耦，用户触发 Dock 时不应同步显示顶部菜单栏。
- Esc 必须可以退出全屏界面。
- 再次触发全局快捷键必须可以隐藏界面。
- 点击空白模糊背景可以隐藏界面。
- 点击应用启动后必须隐藏界面。

视觉目标：

- 遵守 Apple 设计语言，参考 Apple Human Interface Guidelines。
- 使用系统材质、系统字体、系统图标、克制层级和可预期交互。
- 全屏背景默认呈现当前系统背景的模糊处理，不使用纯灰色或强装饰背景。
- 默认界面只保留液态玻璃风格搜索框、弱化设置入口、应用网格和分页点。
- 默认分页网格保持 7×5，图标默认尺寸为 96×96，接近系统启动台视觉重量。
- Dock 图标应接近 Apple 官方 Apps/启动入口图标的语义，使用自有圆角底和 9 宫格 app tile；菜单栏状态项图标保持独立的简洁网格符号。
- 搜索框应位于顶部中间但不能贴近屏幕边缘，应包含放大镜图标、足够内边距和轻量清除按钮。
- 排序、刷新等高级操作应弱化展示，避免破坏系统启动台观感。

参考：

- Apple Human Interface Guidelines: https://developer.apple.com/design/human-interface-guidelines
- Apple HIG Materials: https://developer.apple.com/design/human-interface-guidelines/materials
- Apple HIG Designing for macOS: https://developer.apple.com/design/human-interface-guidelines/designing-for-macos

### 7.9 设置

MVP 设置项：

- 排序方式。
- 排序顺序。
- 显示模式。
- 网格密度。
- 快捷键。
- 重新扫描应用。
- 重置自定义排序。

设置入口：

- 主全屏界面在搜索框右侧提供弱化设置入口，承载排序方式、显示模式、重新扫描等基础操作；退出入口不放在主界面设置菜单中，避免误触。
- 排序入口不增加单独的“排序方式”外层子菜单；排序类型直接出现在设置菜单的排序分组中，添加时间、名称、最近打开右侧展开具体顺序，“自定义”作为直接选项。
- 设置菜单使用 AppKit 原生 `NSMenu` 和少量系统组间分隔线，不使用会吞掉功能图标或插入额外分割线的 SwiftUI `Menu`/inline Picker；使用系统分组标题表达“排序方式 / 显示模式”，重新扫描直接放在下方分隔线之后。
- 主界面设置菜单的选中项应在左侧保留独立勾选列，右侧继续显示功能图标和标题；二级排序中，一级排序类型和二级具体顺序都应显示选中态。
- 菜单栏状态项左键直接进入 OpenLaunch；右键或 Control 点击只保留“重新扫描 / 退出”维护动作。
- 状态栏不承载排序、显示模式等设置入口，避免刘海屏、状态栏图标过多、全屏隐藏策略或菜单图标选中态影响可用性。
- 网格密度、快捷键、重置自定义排序等高级设置后续进入独立设置面板。

### 7.10 命名与本机安装

命名决策：

- 用户可见产品名确定为 `OpenLaunch`。
- app bundle 名、Dock 名称、SwiftPM package/target 名保持 `OpenLaunch`。
- 仓库名、本地目录名可以使用 `open-launch` 这种 kebab-case。
- 不建议把用户可见应用名写成 `open-launch`，它更像命令行包名或仓库名，不符合 macOS 图形应用的常见命名习惯。

本机安装交付：

- 当前阶段提供 `.app`、`.dmg`、`.pkg` 三种本机 Developer Build 产物。
- `.app` 用于快速开发运行。
- `.dmg` 用于拖拽安装到 `/Applications`，推荐作为当前主要本地安装方式。
- `.pkg` 用于命令行或安装器方式安装到 `/Applications`，仅作为开发阶段备选。
- 当前安装包未签名、未公证，仅用于本机开发验证。

## 8. 技术方案

推荐技术栈：

- Swift。
- SwiftUI。
- AppKit。

职责划分：

- SwiftUI：网格 UI、设置界面、搜索、拖拽排序。
- AppKit：窗口层级、全局快捷键、当前屏幕显示、应用启动、菜单栏状态。

窗口架构约束：

- 主启动器窗口必须由 AppKit 显式创建，SwiftUI 只作为 `NSHostingView` 内容层。
- 不使用 SwiftUI `WindowGroup` 承载主启动器窗口，避免系统窗口恢复把 OpenLaunch 还原成普通小窗口。
- 主窗口使用无边框 `NSWindow` 子类，必须允许成为 key/main window，保证搜索输入和 `Esc` 退出稳定。
- 主窗口禁用窗口恢复，构建产物声明 `NSQuitAlwaysKeepsWindows = false`，AppDelegate 不支持 secure restorable state。
- 构建产物不声明 `LSUIElement`；OpenLaunch 覆盖层可见期间保持 `.regular` 前台身份以应用系统栏隐藏策略，覆盖层隐藏后再切回 `.accessory`，尽量保持平时不占 Dock 运行点。
- 显示过程中的激活切换可能触发调用方 app 重新成为前台应用；需短暂忽略该系统回弹，避免启动器刚打开就被外部激活监听隐藏。
- 覆盖层可见期间使用 `autoHideMenuBar` / `autoHideDock` 默认隐藏系统栏，并显示透明顶部触发拦截层，避免鼠标贴顶时菜单栏浮出；隐藏覆盖层时恢复系统状态。
- `hideMenuBar` 只能与 `hideDock` 同时使用，会破坏 Dock 边缘触发能力，因此不作为默认策略。
- 覆盖层可见期间临时隐藏 OpenLaunch 状态栏项，避免菜单栏被系统显示时出现图标或空白占位；覆盖层隐藏后恢复状态栏入口。
- 主内容窗口层级低于系统 Dock、高于普通窗口，避免 Dock 只露阴影或被 OpenLaunch 覆盖；顶部菜单栏不通过窗口遮罩处理。
- Foundation：配置持久化、文件扫描、JSON 数据。
- NSWorkspace：打开应用、获取图标。
- Spotlight metadata / FileManager：读取添加时间与文件属性。

## 11. 风险分析

### 11.1 全屏覆盖体验风险

风险：

- 第三方 app 无法完全复制系统 Launchpad 的窗口层级和手势体验。
- 在全屏 app、多个 Space、多显示器场景下可能有边界问题。

缓解：

- MVP 采用覆盖式全屏窗口，显示在鼠标所在屏幕。
- 后续再评估多屏幕独立布局、跨 Space 动画等高级体验。
- 使用公开 AppKit API，不追求系统级动画。

### 11.2 添加时间不准确

风险：

- macOS 没有统一可靠的安装时间字段。

缓解：

- UI 使用“添加时间”。
- 首次建档校验 Spotlight metadata，失败或异常时降级。
- 持久化首次确认的添加时间，避免自动更新导致顺序漂移。
- 默认旧到新排序，确保新添加的软件出现在最后。

### 11.6 Apple 官方感不足

风险：

- 普通标题栏、过多控件、醒目的第三方工具栏或不符合系统习惯的交互，会让界面明显不像 Apple 官方出品。

缓解：

- 使用 borderless 覆盖式全屏窗口。
- 默认隐藏复杂设置，仅保留搜索、应用网格和分页点。
- 使用系统材质、SF Symbols、系统字体和自然动效。
- 键盘文字输入进入搜索框，但初始进入不主动显示输入光标。
- Esc、快捷键、点击空白背景、滑动翻页、点击分页点必须符合用户对系统启动台的预期。

### 11.3 应用扫描噪音

风险：

- 系统中存在 helper app、后台工具、重复 app。

缓解：

- 根据 Info.plist 过滤 `LSBackgroundOnly` 后台类型，保留有 UI 或菜单栏入口的 `LSUIElement` 应用。
- 默认只扩展到 `/System/Library/CoreServices/Applications` 这类系统公开应用位置，不把全部 `/System/Library/CoreServices` agent 纳入启动器。
- 候选 URL 先按标准化路径去重，再按 bundle identifier 去重。
- 提供隐藏应用能力作为后续功能。
- MVP 先做基础过滤，避免复杂规则。

### 11.4 性能风险

风险：

- 应用数量多时，图标加载可能慢。

缓解：

- 首次扫描和图标缓存异步执行。
- 主界面优先显示文本/占位图标。
- 缓存图标缩略图。

### 11.5 快捷键冲突

风险：

- 默认快捷键可能与系统或其他 app 冲突。

缓解：

- 默认使用低冲突快捷键。
- 设置页允许修改。
- 冲突时给出提示。

## 13. 验收标准

MVP 验收标准：

- 应用启动后能扫描并展示本机 app。
- 点击任意正常 app 可以启动。
- 启动后 OpenLaunch 自动隐藏。
- 名称排序稳定可用。
- 添加时间排序可用，并能处理缺失时间；默认旧到新，新软件在最后。
- 最近打开排序可用。
- 自定义拖拽排序保存后重启仍有效。
- 分页模式和滚动模式都可切换。
- 分页圆点可点击跳页，横向滑动可翻页。
- 搜索可按名称过滤，输入实时进入搜索框。
- 搜索中文展示名和英文原名均可命中。
- 新增应用在下一次打开启动台时有一次性入场动画。
- Esc 可退出全屏界面。
- 快捷键可呼出和隐藏窗口。
- 重新扫描不会造成崩溃。
- 删除某个 app 后，OpenLaunch 能在下次扫描后更新列表。

非功能验收：

- 冷启动后 2 秒内显示主界面。
- 365 个左右 app bundle 的机器上，滚动保持流畅。
- 图标和文字不重叠。
- 无需网络即可工作。
- 不使用私有 API。

## 16. 推荐决策

推荐 MVP 决策：

- 产品名：OpenLaunch。
- 默认显示模式：分页。
- 默认排序：添加时间，方向为旧到新，新软件在最后。
- 搜索框：默认显示。
- 菜单栏状态项：加入，便于打开启动台、重新扫描和退出。
- 状态栏图标：参考 LaunchOS / lanuchos 的简洁启动器气质，采用 OpenLaunch 自有的网格/启动符号设计，避免直接复制。
- 隐藏 app：延后到 v0.2。
- 多显示器：MVP 先显示在当前主屏幕，v0.2 支持鼠标所在屏幕。
- 发布方式：先本机 Developer Build，功能稳定后再做签名和公证。
